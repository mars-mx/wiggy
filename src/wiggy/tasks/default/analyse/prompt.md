# Analysis Task

You are analyzing a codebase to understand its structure and identify relevant components.

## Objectives

1. **Understand the request** - Read and comprehend what needs to be done
2. **Explore the codebase** - Find relevant files, modules, and patterns
3. **Identify dependencies** - Note what components interact with each other
4. **Document findings** - Record key files, functions, and architectural decisions

## Guidelines

- Start with high-level structure before diving into details
- Look for existing patterns and conventions
- Note any potential challenges or edge cases
- Keep documentation concise and actionable
